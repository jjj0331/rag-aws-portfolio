# rag-aws-portfolio
RAG（AIエージェント）を使用したG検定特化の学習アプリケーション開発

## どんな課題がある
G検定を学習する上で、ChatGPTを使用しているのだが、ChatGPTの回答は幅が広く、現在活用している参考書の内容と離れたニッチな回答が多い

## 解決方法
G検定の参考書の内容をDBに保存、APIを叩き、DBの情報を元に回答を生成
また、ChatGPTの回答は長すぎる傾向にあるため、DBには模範回答を用意く、
さらに、G検定の出題範囲の用語は多く、参考書をいきなり読むことは難しいため
回答内容は中学生にも分かるように噛み砕いて回答する工夫。

### バックエンド

・ベクトルDB
・要素集：書く用語を中学生にも分かるように、一言で書いたもの

### 📲 アプリの動作フロー（設計図）
User: 「ディープラーニングって何？」と質問する。
Frontend: POST /api/v1/query でBackendに質問を投げる。
Backend:
(a) 質問をベクトル化し、ベクトルDB (A) を検索。「ディープラーニングとは...」という専門的な資料（カンペ）を見つける。

(b) このカンペと質問をOpenAI APIに渡し、「この資料を元に、G検定の受験者向けに回答して」と指示する。

(c) AIが「ディープラーニングは、人間の脳を模したニューラルネットワークの一種です」という回答（answer）を生成する。

(d) Backendは、その回答（answer）をスキャンし、「ディープラーニング」「ニューラルネットワーク」という単語を辞書DB (B) で検索する。

(e) 「中学生レベルの注釈」を見つける。
Frontend: Backendから、answerとannotationsの両方を受け取り、画面に表示する。
